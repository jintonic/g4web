# Security Policy

## Supported Versions

We actively provide security updates for the following versions of this project:

| Version | Supported |
| ------- | --------- |
| 1.0.x   | ✅ Yes    |
| < 1.0   | ❌ No     |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security-related items.**

If you discover a potential security vulnerability in this project, please report it by following these steps:

1. Email us at **<jing.liu@usd.edu>**.
2. Include a detailed description of the vulnerability.
3. Provide steps to reproduce the issue (proof-of-concept scripts/screenshots are helpful).

### Our Response Process

- **Acknowledgement:** You will receive a response within 48 hours acknowledging your report.
- **Investigation:** We will investigate the issue and keep you updated on our progress.
- **Fix & Disclosure:** Once a fix is ready, we will coordinate a release and public disclosure with you to ensure users are protected.

## Preference for Private Reporting

If your repository is part of a GitHub Organization, you may also use the **"Report a vulnerability"** button under the "Security" tab of this repository to start a private advisory.
